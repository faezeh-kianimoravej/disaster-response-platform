## Definition of Done
- The feature is developed according to requirements and design.
- The feature has been validated with a FAT (Functional Acceptance Test) or feature-based testing.
- The feature builds without issues in DevOps and passes vulnerability checks (build pipeline validation in a pull request).
- The feature has no merge conflicts with the Dev branch.
- The feature has been accepted in a pull request.
- (Optional) Automated tests are executed successfully.
